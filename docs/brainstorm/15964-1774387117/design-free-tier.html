⚠️ Problema trovato e risolto: Cron Job sessione
❌ Non funziona su Free
Vercel Hobby: max 2 cron jobs, frequenza minima 1 volta al giorno
→ "controlla ogni minuto se la sessione è scaduta" = impossibile su free
✅ Soluzione: 3 livelli gratuiti
1. Ogni chiamata GPS (ogni 10s) controlla se now() ≥ end_at → chiude automaticamente
2. Supabase pg_cron (gratis, dentro Postgres) ogni 1 min come fallback
3. Admin dashboard polling ogni 30s → chiude se scaduta
In pratica: la sessione si chiude entro max 10 secondi dallo scadere del timer (al prossimo aggiornamento GPS di qualsiasi giocatore). Supabase pg_cron garantisce la chiusura anche se tutti i giocatori sono fermi. Zero Vercel Pro richiesto.

⚠️ Realtime 200 connessioni simultanee — coincide esattamente con il limite massimo dell'evento (200 giocatori). Se si supera: non tutti i giocatori ricevono aggiornamenti leaderboard live. Soluzione: leaderboard live solo per admin dashboard (1 canale); giocatori ricevono aggiornamenti ogni 30s via polling HTTP normale invece di WebSocket. Duelli rimangono realtime (2 connessioni per duello, priorità alta).
⚠️ Pausa Supabase dopo 7 giorni inattivi — il DB va in pausa se non ci sono query per una settimana. Soluzione: un semplice Vercel Cron (1/giorno, gratis) fa una query keep-alive. Oppure aggiornare a Supabase Pro ($25/mese) solo nel periodo degli eventi.
