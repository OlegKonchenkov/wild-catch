Design Corrections — Applied

4 important updates to the design based on your feedback

🇬🇧
1. App language: English throughout
All UI labels, button text, system messages in English. Italian only appears in admin-entered content (creature names, story text) — whatever the admin writes. Screen names: Map, Bestiary, Missions, Backpack, Profile.
📍
2. Map = small defined area, not the whole city
The map shows only the event's bounding box (e.g. a park, a piazza, a museum grounds). Admin sets the bounding box when creating a session. Outside the area → encounters stop, player sees "You're outside the event area" message. Map tiles are OpenStreetMap cropped to the bounds — works at any location worldwide, not just Pesaro.
📖
3. Story / narrative is fully admin-configurable
The app has no hardcoded story. The admin sets per session:
  • Event name & tagline (e.g. "WildCatch: The Lost Key")
  • Story intro text (shown at onboarding — written by admin, can be any theme)
  • Chapter titles & descriptions (the escape room arc)
  • NPC names (villain, arena boss, ally character)
  • Mission flavor text (each mission has admin-written narrative)
The UI renders whatever narrative content the admin enters. This session could be a Pokémon-themed escape room. Next session could be a pirate adventure. The game engine is theme-agnostic.
🔧
4. How this works technically
A sessions table stores all narrative metadata as structured JSON fields:
{
  "story_title": "The Lost Key",
  "story_intro": "A mysterious creature has stolen...",
  "villain_name": "Shadow Master",
  "chapters": [
    { "title": "The First Clue", "description": "..." },
    { "title": "The Hidden Chamber", "description": "..." }
  ],
  "area_bounds": { "sw": [...], "ne": [...] }
}
Admin fills this in a rich text form. Players see it rendered in the Missions screen.
